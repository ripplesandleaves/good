# Contributing to good

Thank you for your interest in contributing.

## What we're looking for

- New skills that codify high-impact best practices in software development
- Improvements to existing skills: sharper principles, better examples, clearer language
- Fixes for inaccuracies or outdated guidance

## What we're not looking for

- Exhaustive coverage for its own sake — each skill should stay focused on the highest-impact principles
- Rewrites of well-working skills for stylistic preference

## How to contribute

1. **Open an issue first.** Describe the principle or change you want to add and why it matters. This saves effort if the direction isn't a fit.

2. **One change per PR.** A PR that adds one clear thing gets reviewed faster than one that changes many things at once.

3. **Follow the skill format.** Every skill file must have:
   - YAML frontmatter with `name` and `description` fields
   - Description starts with "Use when..."
   - Sections: overview, key principles (with examples), red flags
   - No placeholders or TODOs

4. **Plain language.** Read the good-writing skill before writing content.

5. **Apply /good:good-pr** when opening your pull request.

## Skill quality bar

A skill earns its place when:
- It covers principles that are genuinely high-impact, not just good hygiene
- A developer could read it in 60 seconds and immediately apply it
- An AI agent loading it would make meaningfully better decisions
